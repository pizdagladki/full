# 03. Честность исхода и CV-детекция (anti-cheat)

> Зона: всё в этом разделе — `[AGENT]` (механика). Финального визуала здесь нет.

## Архитектура честности

- **CV на клиенте.** Каждый игрок судит СВОЁ моргание локально. Это главный приоритет: детекция не зависит от сети.
- **Серверный арбитраж времени.** Сервер решает «кто раньше» по серверным меткам времени (НЕ по пингу), а не анализирует видео.
- **Буфер подтверждения 100–200мс** перед объявлением проигрыша — оппонент успевает увидеть момент (полный бюджет ~230мс).
- **Killcam отвергнут** (дорого).
- **Уровень анти-чита MVP = C** (серверный арбитраж времени + правил, без видео-анализа на сервере).

## CV-детекция моргания

- **Технология:** computer vision на клиенте — MediaPipe Face Mesh / FaceLandmarker, blink detection по Eye Aspect Ratio (EAR). НЕ LLM: реалтайм покадрово, локально, низкая задержка, без затрат на инференс.
- **Правило игры:** моргание ОДНИМ глазом = проигрыш (как в дворовых гляделках).
- **Резкость детекции:** подтверждение 2 кадра (~67мс), диапазон 2–3, финал на playtest.
- **Анти-ложняк:** персональный порог НА КАЖДЫЙ глаз, сглаживание EAR, гейт по уверенности landmark'ов.
- **Калибровка:** только Уровень 1 (замер открытого глаза), АВТО и НЕВИДИМО — фоном на превью камеры + в 5-сек отсчёте; не удалась → дефолтный порог.

## Адаптация качества под слабую сеть

- Режем РАЗРЕШЕНИЕ, держим fps 24–30 (моргание ~200мс, ниже 15 fps его можно проскочить). Нижний порог 320×240.

## Сквозной гейт лица (на всём матчмейкинге)

- Без лица в кадре поиск не стартует.
- Ушёл из кадра в поиске → стоп + сброс на домашний экран.
- Найден оппонент → санити-чек 2 сек, нет лица → отмена + поражение.
- Граница наказания = старт таймера боя: до старта — мягко (сброс), после старта — увод лица = проигрыш.
- ELO-штрафов в предбое НЕТ.
- Побочный эффект: гейт устраняет дедлок ожидания.

## Осознанный долг

- Подмена клиентского вердикта возможна (нет серверной видео-верификации) — принято сознательно. Проверка кадров на сервере — пост-MVP. В MVP эту дыру закрывают репорты игроков (см. `09-reporting.md`).

## Кольцевой буфер (важная связь)

- Лёгкий кольцевой буфер кадров ~2 сек крутится весь бой. Используется и для буфера подтверждения (этот раздел), и для записи победной части (см. `06-sharing.md`, Пласт 1).
