# 04. Сеть и видео (WebRTC)

> Зона: всё — `[AGENT]`.

- **Топология: P2P** (peer-to-peer). Видео идёт напрямую между браузерами. Обоснование: бой всегда 1-на-1 (главный плюс SFU — много участников — не нужен), сервер видео не анализирует. P2P даёт минимальную задержку и почти не нагружает сервер. Через инфраструктуру идёт ~15% трафика против 100% у SFU.
- **SFU/media-сервер:** НЕ используем (оправдан только при много-участниках или серверной обработке видео).
- **STUN:** берём СРАЗУ (публичные серверы, нужен для установки P2P, бесплатно). НЕ путать с TURN.
- **TURN** (ретранслятор для ~10–20% игроков со строгим NAT / мобильным / корп-сетями): **в БЭКЛОГ** (вторая итерация до MVP, не основной заход). На старте облачный TURN, свой coturn при росте. Важно: coturn — это «тупой» ретранслятор пакетов внутри того же P2P, НЕ SFU.
- **Управляющий канал:** WebSocket (Go, coder/websocket) — signaling для установки P2P (обмен SDP/ICE) + арбитраж времени.

## Осознанный долг

- Без TURN у части тестеров (мобильные/корп-сети/строгий NAT) соединение НЕ встанет — это НЕ баг, а отсутствие фолбэка. Не тратить время на отладку до внедрения TURN.
