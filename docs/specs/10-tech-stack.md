# 10. Технологический стек

## Клиент (фронтенд)

- **Фреймворк:** React (много обычного UI — магазин, профиль, итоги, лендинг; тяжёлая часть — canvas/CV/WebRTC — в изолированных компонентах через ref'ы).
- **CV-детекция:** MediaPipe Face Mesh / FaceLandmarker, EAR (локально, покадрово).
- **Запись эдита:** canvas + MediaRecorder (WebM), captureStream — на лету.
- **Эдиты:** HTML/Canvas/WebGL шаблоны (произведены в Claude Design), рендер поверх видео.

## Сеть / видео

- **WebRTC P2P** (видео между игроками).
- **STUN** сразу; **TURN** (coturn / облачный) — бэклог.

## Бэкенд

- **Go + Echo (`labstack/echo/v4`) + coder/websocket** — REST (Echo); signaling SDP/ICE, серверный арбитраж времени, матчмейкинг (coder/websocket).
- **Конвертация WebM→MP4** (серверная, для соцсетей).

## Данные

- **PostgreSQL** — основная БД (юзеры, рейтинги, история матчей, покупки, инвентарь, репорты). Реляционные данные + надёжные транзакции (важно для денег). Дружит с Go, есть JSONB для гибких полей (напр. мета отвлекалок).
- **Redis** — состояние матчмейкинга (очередь поиска в реальном времени), кэш горячих данных (рейтинги), кулдауны/сессии. Берём сразу.

## Хранилище файлов

- **MinIO** (self-hosted, S3-совместимый) на старте → миграция на S3 при росте (тот же API).

## Платежи

- **Stripe** основной (через внутреннюю абстракцию «платёжный провайдер», чтобы добавить второго не переписывая логику покупок).
- РФ-платёжка (ЮKassa / CloudPayments) — отдельным провайдером позже.
- ⚠ Stripe требует юрлицо/компанию в поддерживаемой стране (РФ не поддерживается как страна мерчанта) — организационный вопрос до запуска.

## Авторизация

- **Google OAuth** (основная).

## Инфраструктура

- **DigitalOcean + Docker** (docker-compose: Go + Postgres + Redis + MinIO + coturn-потом).
- Фронт — статика (CDN / тот же VPS).
- **Kubernetes — пост-MVP** (явно не в первый заход).

## Внешние сервисы

- **AdSense** (баннеры, чистые зоны).
- **AdMob / Unity Ads** (rewarded video).
- **Telegram-бот** (приём багрепортов).
