# 05. Рейтинг (ELO)

> Зона: вся логика рейтинга — `[AGENT]`. Прогресс-бар уровня (визуал) — `[AGENT-SCAFFOLD]` каркас + `[OWNER-DESIGN]` финальный вид и анимация ±ELO.

- **Формула:** классический ELO — Δ = K × (факт − ожидание), где ожидание = шанс победы из разницы рейтингов. Система реагирует на неожиданность: обыграл сильного → много (+~29), обыграл слабого → мало (+~3), проиграл сильному → почти не больно (−~3), проиграл слабому → наказание (−~29).
- **K-фактор:** 64 для новичка (первые ~20 игр) → ускоряет калибровку до реального уровня примерно вдвое; затем стандартный 32.
- **Симметрия — АСИММЕТРИЯ (мягкость к проигравшим):** проигравший теряет 80% от честной потери (в равном матче −12.8 вместо −16). Победитель получает полные очки. Цель — поражение не так больно, лучше удержание.
- **Цена — инфляция рейтинга:** ~+3.2 ELO/матч впрыскивается в систему (~+288 за квартал у активного игрока).
- **Уровни:** 10 шт (L1: 100–500 … L10: 2001+), старт L4 (1000 ELO).

## Бэклог (следующий приоритет): Сезонные сбросы

- Раз в N месяцев (предв. ~3 мес) сезон завершается, рейтинги частично откатываются к центру — инфляция обнуляется.
- Механизм (предв., soft reset): `new = 1000 + (old − 1000) × 0.5`. Пример: 1800 → 1400, 2100 → 1550.
- Бонус: сезоны сами по себе — механика удержания (свежий старт, мотивация, награды/лидерборды — TBD).
- Зависимость: без этого механизма асимметрия со временем топит систему уровней.
