# Гляделки — Спецификация MVP

> Веб-стартап: соревновательная онлайн-игра «гляделки». Двое смотрят друг на друга через камеры, кто моргнул — проиграл. Фишка продукта — победные видео-эдиты под TikTok-трек, которыми можно делиться.

Это разбитая на разделы спецификация MVP. **PM-агент / feeder: начни с этого файла**, затем читай разделы по порядку. Каждый раздел — отдельный `.md`.

## Зоны ответственности (читать ПЕРВЫМ)

Спека режется на таски (feeder → issues). Каждый блок работ помечен тегом, кто его делает. **PM-агент / feeder обязан соблюдать эти теги при нарезке тасок.**

- **`[AGENT]`** — агент-исполнитель делает полностью: бэкенд, логика, структуры данных, механика, интеграции. Готовый результат.
- **`[AGENT-SCAFFOLD]`** — агент строит КАРКАС: структуру экранов, компоненты, роутинг, состояние, рабочую механику на **нейтральном дизайне** (простые плейсхолдер-стили) + **функциональные заглушки анимаций** (напр. для эдита — прямоугольник-слот, играющий 10 сек; для отвлекалки — серый оверлей на N% экрана с правильным таймингом). Всё работает механически, но выглядит «никак».
- **`[OWNER-DESIGN]`** — зона владельца, агент НЕ трогает. Весь финальный визуал: реальные анимации эдитов (делаются в Claude Design → HTML/Canvas/WebGL), реальные визуалы отвлекалок, motion-анимации интерфейса, финальный стиль/брендинг.
- **`[MERGE]`** — отдельная таска слияния: владелец отдаёт готовый визуал/ассеты → вмерживаются в каркас агента. Создаётся ПОСЛЕ того, как owner отдал материалы (агент не ждёт блокирующе, строит на заглушках).

**Принцип:** агент делает так, чтобы ВСЁ работало механически на заглушках; владелец сверху накатывает красоту через `[MERGE]`-таски. Флот НЕ поручается финальный визуал — только каркас + функциональные заглушки.

## Карта осознанных долгов (принято сознательно для MVP)

| # | Долг | Раздел | Митигация / план |
|---|------|--------|------------------|
| 1 | Нет серверной видео-верификации (подмена вердикта возможна) | 03 | Репорты игроков как анти-чит; проверка кадров — пост-MVP |
| 2 | Без TURN часть игроков не соединится | 04 | TURN в бэклог (2-я итерация до MVP) |
| 3 | Инфляция ELO от асимметрии | 05 | Открытый Сезон 1 + сброс при закрытии |
| 4 | Лицо оппонента в шере без полноценного согласия (GDPR/BIPA/несовершеннолетние) | 06 | 3 галки; опц. блюр на будущее |
| 5 | P2W-отвлекалки в рейтинге убивают доверие к ELO | 07 | Контрмеры — след. итерация |
| 6 | AdSense может забанить за живые камеры незнакомцев | 08 | Проверить политики до внедрения |
| 7 | Абуз репортов (незаслуженные кулдауны) | 09 | Защиты от абуза — след. итерация |
| 8 ★ | **Крипто-аирдроп: обещание токена за холд = регуляторный риск (SEC/MiCA), без геоблока США** | 11 | **ЮРИСТ до токен-части** (не «после MVP»); ToS без гарантий; отд. юрлицо-эмитент; токен как utility; анти-абуз перед токеном |

## Сводка отложенного (для следующих сессий)

- **Точная экономика:** цены отвлекалок и платных эдитов; числа начисления поинтов и цены айтемов в поинтах (в MVP — config-плейсхолдеры).
- **Логика реванша:** механизм обоюдного согласия (кнопка «Реванш» на экране итогов).
- **Сезонные сбросы:** полный дизайн механики (следующий приоритет после спеки).
- **Защиты от абуза репортов:** вес репортёров, уникальность, лимиты.
- **Контрмеры против отвлекалок:** «стряхнуть», щит и т.п.
- **Оверлей на лицо (AR-косметика + индикатор статуса):** полностью пост-MVP.
- **Винстрик, дейлики, премиум-подписка:** бэклог.
- **Токен / аирдроп / анти-абуз-перед-токеном / юр-часть:** человеческий пререквизит (см. `11-points-system.md`), НЕ задачи флота.

## Связь с репо и флотом

Спека живёт в `docs/specs/` и потребляется автономным флотом (`/feeder docs/` режет её на задачи). Рядом — инженерный канон репозитория (зона человека, флот туда не пишет):

| Файл | Что это |
|---|---|
| `tech-stack.md` | **Канонический** тех-стек (источник правды по бэкенду — скилл `go-backend-conventions`; при расхождении правит он). Конкретные либы: Echo, coder/websocket, pgx v5, go-redis v9, minio-go, zap, validator/v10. |
| `architecture.md` | Структура монорепо `pizdagladki/full` (services/<name>/, internal/platform/, слоёная архитектура). |
| `local-dev.md` | Runbook: локальный запуск/тесты бэка и фронта, деплой (DigitalOcean + Docker). |
| `fleet-flow.md` | Как работает автономный флот Claude Code: цикл `/work-cycle`, лейблы, гейты, GitHub как шина состояния. |

**Связь зон спеки с feeder (важно при нарезке):**

- `[AGENT]` / `[AGENT-SCAFFOLD]` → обычные задачи флота.
- `[OWNER-DESIGN]` → **человеческий пререквизит**, флот не строит. Feeder помечает `Manual prerequisite (human): …`, задачи не создаёт.
- `[MERGE]` → задача, заблокированная до тех пор, пока владелец не отдал ассеты/дизайн (`Depends on` владелец-deliverable).
- **Токен / аирдроп / юр-часть / анти-абуз-перед-токеном** (раздел 11) → тоже человеческий пререквизит, НЕ задачи флота. В MVP флот строит только earning/spend/hold/виджет.
- `10-tech-stack.md` — продуктовый срез; при любом конфликте по бэкенду побеждает канон `tech-stack.md` + скилл `go-backend-conventions`.
