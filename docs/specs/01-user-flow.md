# 01. User Flow (путь пользователя)

> Зоны: фронт-каркас всех экранов — `[AGENT-SCAFFOLD]` (нейтральный дизайн); финальный стиль — `[OWNER-DESIGN]`.

1. **Лендинг** — название проекта, инфо, кнопка авторизации.
2. **Регистрация** — 3 ОТДЕЛЬНЫЕ ГАЛКИ (granular consent, нормальным шрифтом, отдельными документами, НЕ закопано в общий ToS):
   - подтверждение совершеннолетия (18+);
   - согласие на запись и публикацию своего лица в видео батлов;
   - полное пользовательское соглашение.
3. **Авторизация** — Google OAuth (основная).
4. **Домашний экран:**
   - подготовка к батлу: выбор камеры, выбор TikTok-трека для эдита;
   - магазин (отвлекалки + эдиты);
   - профиль, рейтинг;
   - ПРОГРЕСС-БАР УРОВНЯ виден здесь (не только в итогах);
   - идёт фоновая авто-калибровка на превью камеры;
   - AdSense-баннеры.
5. **Выбор режима:** безранговый / рейтинговый / пригласить друга.
6. **Вход в матчмейкинг.** СКВОЗНОЙ ГЕЙТ ЛИЦА: без лица в кадре поиск не стартует; ушёл с камеры в поиске → стоп + сброс на домашний экран.
7. **Анимация поиска.** Бэкенд подбирает соперника по уровню (фолбэк ~10 сек на ближайший уровень).
8. **Найден оппонент** → P2P-соединение (WebRTC) → Battle Flow (см. `02-battle-flow.md`).
9. **После боя:** эдит победы → экран итогов (результат, ±ELO с прогресс-баром, длительность, кнопки реванш / играть снова, шеринг в TikTok, репорт).
10. **Профиль:** статистика, история боёв, 10 сохранённых моментов (просмотр / скачивание / повторный шер).

## Зоны ответственности

- `[AGENT]` — логика авторизации (Google OAuth), сохранение галок согласия с датой, навигация/состояние, гейт лица (механика).
- `[AGENT-SCAFFOLD]` — структура всех экранов, компоненты, роутинг, нейтральные стили, анимация поиска (функциональная заглушка).
- `[OWNER-DESIGN]` — финальный стиль/брендинг всех экранов, реальный motion (анимация поиска, переходы).
- `[MERGE]` — вмерж финального дизайна в каркас.
